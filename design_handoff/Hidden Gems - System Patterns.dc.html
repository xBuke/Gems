1
4 toast types — left border = severity
Teal = success, Red = error, Amber = warning, Teal info = system. Left border is the only colour — body stays on-theme (card bg). Toasts slide down from top, auto-dismiss after 4s. Error toasts stay until dismissed or action taken.
2
Offline = amber sticky banner
Offline state gets a persistent amber banner at the top of every screen — not a toast (which would auto-dismiss). App continues to show cached content. Banner disappears automatically when connection resumes.
3
Inline errors use dot + SpaceMono
Field errors: red border on input + small dot + SpaceMono error text below. No floating tooltips. Error clears on first keystroke (onChangeText). Full-screen errors only when no data can be shown at all.

1
Tabs never slide — always crossfade
Tab switches crossfade (200ms). Sliding between tabs implies an ordered sequence — these tabs are peers, not a sequence. Stack pushes slide horizontally (300ms easeInOut). Modals spring up from below (400ms).
2
Shared element on gem card → detail
Use Reanimated.SharedTransition on the card image and title. Tag with gemCard_{"{id}"} so each card gets a unique transition. The category badge and coordinates animate in separately after arrival.
3
Reduce motion: respect system preference
Wrap all transitions in AccessibilityInfo.isReduceMotionEnabled(). If true: crossfade only — no slides, no shared elements, no springs. Duration: 150ms max.

1
3 notification categories
Nearby gem (location-triggered), social (follows/likes/comments), achievements. Each has its own icon treatment. Android notification icon = monochrome compass PNG. Set channelId per category for user control.
2
In-app banner replaces system push
When the app is foregrounded, intercept with addNotificationReceivedListener and show an in-app banner instead (slides down from top, 4s auto-dismiss). Same content as the push, styled with the app's card + theme tokens.
3
Deep links from tap
Nearby gem → Gem Detail. Follow → Profile (other user). Achievement → Profile own, achievements tab. All via addNotificationResponseReceivedListener + router.push().

1
Search across 3 entity types
Gems, people, and places in one results list — labeled with SpaceMono type tags (GEM / PERSON / PLACE). Filter pills narrow to a single type. "All" is default. Results are ranked by match score × social graph proximity.
2
Query term highlighted in teal
Matched portion of name/title renders in theme.accent. Use a simple HighlightText component that splits on the query and wraps matches in a tinted span — no regex needed.
3
Recent searches persist + clearable
Last 5 searches stored in AsyncStorage and shown as chips when the search bar is focused but empty. Each chip has an × to remove. "Clear all" link above the row.

1
@gorhom/bottom-sheet for all sheets
One implementation for every bottom sheet in the app — action sheets, filter sheets, Add Gem, paywall. Custom handleComponent renders the teal drag bar. overScrollMode="never" on iOS.
2
Map sheet has two snap points
Map detail sheet snaps at 30% (peek) and 65% (expanded). At 30%, only the gem name + category badge + check-in button are visible. Dragging up reveals description + comments. The map remains interactive behind the sheet at 30%.
3
Destructive actions always red
"Report gem", "Delete gem", "Leave community" always render in #F87171 at the bottom of the action list — never inline, always last, always with a confirm step before executing.
