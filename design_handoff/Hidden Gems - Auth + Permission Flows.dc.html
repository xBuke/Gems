1
3-step progress bar
Thin 3-segment bar fills step by step — sets expectation without overwhelming. Active segment = accent, pending = border. Never show a spinner on this screen.
2
Password strength indicator
4-segment bar + label (Weak / Fair / Strong / Very strong). Fills as user types, uses zxcvbn score. No tooltip — inline below the field.
3
SSO above email/password on sign-up
On sign-in, SSO is secondary. On sign-up, we show email/password first (keeps control) with SSO options below the divider — fewer steps for the user who wants to own their credentials.

1
Initial avatar = first letter + coral
Before photo upload, the avatar defaults to the user's first initial on a coral background — same pattern used everywhere in the app. No grey silhouette. If the user skips, this persists as their default.
2
Skip is always available
Never block onboarding on a profile photo. "Skip for now" completes registration and proceeds to the app. Camera permission is requested only when "Take a photo" is tapped — not upfront.

1
Never reveal account existence
The security note explicitly states we won't confirm whether an email is registered — prevents account enumeration. Same success screen shown whether or not the email exists.
2
Countdown resend timer
"Resend in 0:58" counts down — prevents spam. Link after 60s becomes active teal. "Open email app" deep-links to the default mail app via mailto: — one tap from screen to inbox.

1
Ask late, not at launch
Location: asked on first Discover load. Camera: asked on first photo tap. Notifications: asked after first gem saved. Never stack multiple requests — each is shown only at the moment it adds clear value.
2
In-app explain before OS dialog
This screen always appears first. "Allow" triggers the OS system dialog. If the user denies the OS dialog, show a follow-up screen linking to Settings. The OS dialog can only be shown once — this screen primes acceptance.
