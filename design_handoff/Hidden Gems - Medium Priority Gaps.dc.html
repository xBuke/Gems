1
3 cluster sizes, same component
12+ gems = 52px teal filled. 4–11 = 42px slightly smaller. 2–3 = 36px teal-outline. Single gem = coral diamond pin. All use the same MapPin component with a count prop. Tap the 12-cluster above to preview.
2
Tap cluster → thumbnail strip + zoom CTA
Tapping a cluster shows a floating card with up to 3 gem thumbnails + a "+N" tile. "Zoom in to separate pins" is the only action — it increments the map zoom level so pins split apart. Uses react-native-maps clustering props.
3
Outer ring = proximity halo
Large + medium clusters get a 30% opacity ring at 1.3× radius — signals "there are more gems just outside this dot." No ring on small clusters to avoid visual noise at a crowded zoom level.

Muted · Unmute
Muted users' content stays in the feed but is collapsed to a single dimmed row — the user can choose to expand it. Block removes entirely. Both are reversible from Settings → Blocked/Muted accounts.
1
Block vs Mute hierarchy
Mute = soft, reversible, invisible to the other person — their content collapses. Block = hard, bidirectional — both accounts can't see each other at all. Blocked profile shows a generic "not available" screen, not the real username.
2
Muted = collapsed, not removed
A muted user's gem appears as a single dimmed row with "Unmute" — the user is in control. This mirrors Mastodon/X's approach. Never silently hide content — users should always know they can unmute.

Regular members see Feed + Chat + About. Creators also see a "Manage" tab with join requests, pinned posts, member list + actions, and community settings. The "CREATOR" badge appears on their community card too.

1
4 states — one component
LoadMore component takes a status prop: "loading" / "end" / "error" / "idle". Renders the right state. Used as ListFooterComponent on every FlatList in the app — Discover, Notifications, Search results, Profile gems.
2
"You've seen all N gems" — not a blank end
The end-of-list state uses a divider line + SpaceMono copy that tells the user the exact count. Never just a blank gap — that makes users think the app is broken or still loading.

1
Context banner anchors motivation
The teal "BROUGHT YOU HERE" banner with the gem name and pin icon reminds the user why they installed the app. The CTA becomes "Sign up + see gem →" — the gem is the reward, sign-up is the gate. Dramatically increases conversion vs generic sign-up.
2
SSO first for link-installed users
Normal sign-up shows email form first. Deep-link installs show SSO first — these users are high-intent and want the fastest path to the gem. Apple/Google SSO = one tap to account creation. Email still available below the divider.
3
Post sign-up → gem detail directly
After account creation, skip the normal onboarding flow and navigate directly to the gem detail. Store the onboarding state as "completed from deep link" — the category picker and follow suggestions are shown at next app open, not immediately.
