1
Unverified ≠ blocked
User can browse the app while unverified — verification gates posting a gem and messaging. A persistent soft banner appears at the top of their profile until verified. Don't wall off the whole app.
2
Link opens app via Universal Link
The email link is hiddengems.app/verify?token=… — a Universal Link that opens the app directly to the "verified" confirmation screen. Token validated on the server before showing it.
3
60s resend countdown
Same pattern as forgot password — "Resend in 0:47" counts down, becomes an active teal link at 0. "Change email" re-routes to auth step 1 pre-filled so they can correct a typo without losing their progress.

1
Full black background — always dark
Photo viewer is always #000 regardless of theme — photos need full attention without any UI chrome. Status bar: light style. Closes on swipe down (velocity-based). Use react-native-image-viewing or Reanimated.
2
Dot + pill indicator
Active dot expands to a teal pill. "2 / 4" count in SpaceMono top-centre. Swipe left/right to navigate. Top-right: save (bookmark) + share. Both 44×44px targets with semi-transparent dark bg pill.
3
Gem info persists at bottom
Category badge + name + attribution stay visible at bottom of every photo — user always knows which gem they're viewing. Tap the gem name to go to gem detail. Info fades on pinch-to-zoom.

1
Podium + list hybrid
Top 3 get a bar-chart podium with gold/silver/bronze rank badges. Ranks 4+ are a flat list. "YOU" row is always visible, teal-tinted, pinned to the bottom of the list — even if you're rank 200. Score formula shown as a legend.
2
This week vs All time
Weekly resets every Monday 00:00 UTC. "This week" rewards consistent activity and gives new users a fair shot. "All time" satisfies power users who want permanent status. Both tabs remember last position.

1
App stays useful without location
When location is denied, Discover shows globally popular gems instead of a dead empty state. The amber banner explains why nearby isn't available and offers a single "Open Settings" CTA. No distance shown on cards.
2
"Open Settings" uses Linking.openSettings()
One tap opens the app's Settings page directly. iOS: Linking.openURL('app-settings:'). Android: IntentLauncher.startActivityAsync. Banner disappears on app foreground if permission is now granted.
3
Same pattern for camera + notifications
Camera denied → Add Gem shows a "Camera access needed" banner where the photo slot is, with "Open Settings." Notifications denied → Profile shows a soft banner. All use the same amber banner component.
