1
Active = pill + accent, not underline
Active tab gets a 44×32 pill in accentSub behind the icon. No underline bar — it looks too "browser tab." Inactive tabs show only the icon at 50% opacity. Tap each tab above to switch.
2
Add FAB floats above the bar
The coral Add button is 52px, set to marginTop:-14 so it floats 14pt above the bar line. No active state — it always opens the Add Gem modal regardless of current tab. Never highlighted as "active."
3
Badge: count dot, not text-only
Badge sits top-right of the pill, 14px circle, #F87171 (semantic red, stays same in dark/light). Shows count for unread notifications on Profile, new matches on Swipe. Clears on tab visit. Max "99+" for large counts.

HIGH
iPhone 14 Pro+ has a 59pt top inset vs 47pt on standard iPhones. Map floating controls (location button, zoom) need to respect this — currently they may sit 12pt too high.

// Never hardcode top: 47 — always use insets

Use {""}

MED
Map location FAB and Gem Swipe action bar need bottom padding to clear the home indicator on notchless iPhones.

1
4 high-priority iOS items
Dynamic Island insets, status bar style, bottom sheet overscroll, keyboard avoiding — all 4 are one-liner fixes in the right place. Highest risk if missed: status bar shows wrong colour on dark gem detail screens.
2
Always useSafeAreaInsets()
No hardcoded pixel values for top/bottom insets anywhere in the codebase. Use the hook from react-native-safe-area-context — it covers Dynamic Island, notch, and home indicator automatically.
04 — Platform
Android Platform Gaps
Status bar background
HIGH
Android status bar background defaults to black — doesn't inherit the screen colour. Must be set explicitly per screen.
<StatusBar
  backgroundColor={theme.background}
  barStyle="light-content"
  translucent={false} />
Set in app/_layout.tsx globally, override per-screen where the bg differs (map = #0B1A1C, auth = same).
System nav bar (Android 10+)
HIGH
Gesture navigation bar at bottom defaults to white — jarring against the dark tab bar. Use expo-navigation-bar to match.
import * as NavigationBar
  from 'expo-navigation-bar';

NavigationBar.setBackgroundColorAsync(
  theme.background
);
Back gesture on modals
HIGH
Android hardware back on modals crashes or does nothing without onRequestClose. Required for: Add Gem, Paywall, Achievement sheet, Filter sheet.
<Modal
  onRequestClose={handleClose}
  // Add Gem: confirm dialog if dirty
>
Elevation vs shadow
MED
iOS shadow props (shadowOpacity, shadowRadius etc.) are invisible on Android. Cards need elevation too.

shadowOffset: {"{0,4}"}

TouchableOpacity shows no feedback on Android. Use Pressable with android_ripple on all primary CTAs and list rows.

1
3 high-priority Android fixes
Status bar bg, nav bar colour, and modal onRequestClose. All three are one-liners. Without them the app looks unfinished on Android and fails Play Store review guidelines for back-gesture handling.
2
Elevation + ripple: do both
Add elevation:2 to every card that has iOS shadow props. Replace TouchableOpacity with Pressable + android_ripple on all list rows and primary CTAs — swap in one component so the change is global.
3
5 missing haptics — 5 lines of code
Follow accept, community join, trial start, check-in, toast dismiss — all need hapticSuccess/Medium added. Each is one line in the relevant onPress handler. Zero design changes needed.
