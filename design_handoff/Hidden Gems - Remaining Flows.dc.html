1
Same form as Add Gem, pre-filled
Reuses the Add Gem component with existing data pre-filled. Header changes Cancel/Edit Gem/Save instead of ×/Add/Post. Opened from gem detail's ··· menu → Edit.
2
Delete at bottom, confirm dialog required
Delete gem is a red text link at the very bottom — visually separated by a divider. Tapping shows a confirmation bottom sheet ("Delete this gem? This cannot be undone." with red confirm + cancel). Never a single tap to delete.
3
Visibility toggle — new field
Public / Friends / Private segmented control — not in Add Gem (defaults to Public). Only shown in edit so new gems are public by default, discoverable from day one.

Cover renders live from name
The cover preview at the top updates live as the name is typed — same gradient + name overlay as the community list card. "Change cover" lets the user pick a photo or a theme colour.

Open vs Invite only
Open: anyone can join. Invite only: creator approves requests — Join button on the community card becomes "Request to join." Default is Open to maximise discovery.

1
Push vs in-app granularity
Each notification type shows its delivery channel (Push + in-app / in-app only / push only) as a sub-label. A single toggle controls both channels together — advanced users can dig into notification channels in OS settings.
2
Delete account: multi-step with cooldown
Tapping "Delete account" opens a sheet: step 1 explains what's deleted, step 2 asks them to type "DELETE", step 3 shows a 30-day recovery window. Account is deactivated immediately, deleted after 30 days.
