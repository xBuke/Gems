1
Unread = dot + bold + accent time
Unread row: teal dot on avatar, fontWeight:700, accent-colored timestamp, full-brightness message preview. Tap Ana's row to mark read — all four signals clear at once.
2
Message requests quarantined
Strangers' messages are separated into a dimmed "MESSAGE REQUESTS" section — not silently inserted into the main feed. Same pattern as Instagram DMs. Privacy-first default.
3
SpaceMono timestamps + official badge
Timestamps in SpaceMono — consistent with every other time label. The Official account gets the same teal badge pattern used in admin, so users recognise verified senders.

Follow + Message hierarchy
Follow = accent-filled primary CTA. Message = border-only secondary. Post-follow, "Follow" flips to a muted "Following ✓" pill — tap the button above. Own profile shows "Edit Profile" instead; the button slot never disappears.

Stats are tappable links

Read-only gems grid
3-col grid with no edit/delete affordance — long-press context menu is gone. "+42" count tile at end routes to a paginated list. Own profile adds a "+" tile at position 0 as an add shortcut.

1
"Follow Back" tertiary state
Followers tab shows three button states: "Follow Back" (accent-filled, they follow you), "Following" (muted, mutual), "Follow" (outline, new discovery). Priority is visually clear. Tap each to toggle — mutual followers gain the teal MUTUAL badge.
2
MUTUAL badge on rows
Teal "MUTUAL" chip at a glance shows reciprocal follows — no need to cross-reference. Identical pattern to the "OFFICIAL" badge; one shared component, two semantic values.
3
"YOU" highlighted row
In the Following tab, the viewing user's own row is teal-tinted with a "YOU" badge — orientation anchor so you never wonder "am I on someone else's list?" Switch tabs above to see both lists.

1
Gem-share post type
A post where a user shares a gem card is a distinct type — mini gem card embedded inline, not a plain link. Uses the same card component from Discover/Trip Planner. Tap Feed → Chat above to switch views.
2
Category emoji reactions
Reactions use contextual emojis (🔥🌊❤) not a generic 👍 counter. Matches the app's category language and feels native to the explorer community. Available set defined in REACTION_TYPES constant.
3
Chat bubbles: teal for "me"
Own messages use accentSub + accent border — consistent with selected states everywhere else. Others' messages use card — same contrast, different identity.

1
Check-in confirmation sheet
Bottom sheet slides up with compass motif, coordinates, and "check-in #N" count — confirmation that the action registered. Tap "Check In Here" above to see it; "Done" dismisses. Needs hapticSuccess() on trigger.
2
Comment input always visible
The composer bar is sticky at the bottom of the screen — not hidden behind a "Comment" button. Lowers friction for contribution. Avatar in composer personalises the row. KeyboardAvoidingView needed here (see platform gaps).
3
Coordinates on check-in
SpaceMono coordinates + sequential "check-in #25" counter in the sheet reinforce the explorer-data identity. The counter is a social signal: "25 people have stood exactly here." Requires Expo.Location at check-in time.
